Modeling
XGBoost Regression

This system utilizes an XGBoost model optimized for multi-output temperature forecasting

The model was trained on high-resolution data points sourced from the FMI Open Data portal.

Infrastructure
WFS & MongoDB

Observations are fetched via WFS requests. Feature engineering occurs in a Python worker.

Historical persistence is managed by MongoDB Atlas to ensure robust handling of sensor gaps.

Engineering
Predictor Features
  • Temp Rolling Mean: Identifies thermal inertia.
  • Pressure Delta: Signals low-pressure fronts.
  • Temporal Encoding: Sine/Cosine 24h mapping.
Pipeline
GitHub Actions

An automated worker runs hourly via GitHub Actions. It validates sensor data and performs inference.

The cycle concludes by upserting the newest results to the cloud database.
